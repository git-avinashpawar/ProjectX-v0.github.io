Distributed Map-Reduce based Search Engine

View on Github

The Map-Reduce based Search Engine is a powerful search solution that delivers lightning-fast search results. Leveraging the Map-Reduce paradigm, it efficiently processes and queries thousands of Project Gutenberg books. The system utilizes Google Cloud Functions to mine and extract relevant information from the books, enabling quick and accurate search functionality.

The project comprises several key components to facilitate the search engine's operations. The Mapper function plays a crucial role by taking input in the form of book links, downloading the books from the internet, and performing the map() operation on the data. The processed data is then stored in a mapper.json file and uploaded to a shared cloud bucket. The Mapper function is implemented using Node.js.

The Reducer function takes the mapper.json file as input, downloads it from the shared disk, and performs the reduce() operation on the data. The results are stored in a reducer.json file, which is then uploaded to the shared cloud bucket. Similar to the Mapper function, the Reducer function is also developed using Node.js.

The controller acts as a communication hub between the Mapper and Reducer functions. It is responsible for invoking both functions and orchestrating the merge of intermediate outputs from the map() and reduce() operations during each iteration. The controller is implemented using Node.js as well.

The project includes a web user interface (UI) that serves as the endpoint for the search engine system. The UI enables users to interact with the search engine, input their search queries, and display the search results. The web UI also invokes the cloud functions and facilitates the smooth functioning of the search engine. Node.js is utilized for implementing the web UI component.
